== Основы

Простейший случай использования `radioButtonGroup` - выбор значения перечисления для атрибута сущности.

image::visual-components/components/radio-button-group-basics.png[align="center", width="211"]

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/radiobuttongroup/radio-button-group-view.xml[tags=data;userDc;layout;basics]
----

В приведенном выше примере сущность `User` имеет атрибут `onboardingStatus` типа `OnboardingStatus`, который является перечислением.

[[data-binding]]
== Привязка данных

++++
<div class="jmix-ui-live-demo-container">
    <a href="https://demo.jmix.ru/ui-samples/sample/radio-button-group-dataaware" class="live-demo-btn" target="_blank">ДЕМОНСТРАЦИЯ</a>

`radioButtonGroup` может получить список элементов из контейнера данных с помощью атрибута `itemsContainer`. Например:

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/radiobuttongroup/radio-button-group-view.xml[tags=data;hobbiesDc;layout;data-aware]
----

В этом случае `radioButtonGroup` будет отображать имена экземпляров сущности `Hobby`, расположенных в контейнере данных `hobbiesDc`, а его метод `getValue()` будет возвращать выбранный в данный момент экземпляр сущности.

[[item-enabled-provider]]
== Настройка активности элементов

`itemEnabledProvider` позволяет вам динамически управлять состоянием включения отдельных радиокнопок в зависимости от конкретных условий. Отключенные элементы отображаются серым цветом и не могут быть выбраны пользователем. По умолчанию все элементы включены.
