== 状态

`checkbox` 有以下几种状态，不同的状态具有不同的外观和功能。

[[enabled-state]]
*启用*

* 启用：复选框可进行交互并响应用户的输入。用户可以通过点击改变组件的状态。
* 禁用：复选框禁用，无法与用户交互。通常显示为灰色，表示已禁用。

image::visual-components/components/checkbox-enable-disable.png[align="center", width="217"]

====

----
====

[[read-only-state]]
*只读*

`readOnly` 属性控制用户是否可以更改组件的状态。如果设置为 `true`，则 `checkbox` 为只读状态，用户无法更改状态。

.XML 代码
[%collapsible]
====
[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/checkbox/check-box-view.xml[tags=checkbox-readOnly]
----
====

[[indeterminate-state]]
*不确定态*

`indeterminate` 属性控制 `checkbox` 是否处于一个不确定的状态，既非全选中，也非不选中。

在外观上，组件展示为一个单独的不确定态。

NOTE: 根据 HTML5 标准，这个状态只是影响组件的外观，并不会影响组件的值。

== 点击事件

`com.vaadin.flow.component.ClickEvent` 当用户点击组件时发送。处理方法需要指定下列三种类型的点击之一。

* click - 点击时触发

* singleClick - 在一定的延时后触发，以确保不是双击。

* doubleClick - 当双击组件时触发。

== XML 属性

在 Jmix 中，所有组件都有一些作用一致的 xref:vc/common-attributes.adoc[通用属性]。
下面是 `checkbox` 的特殊属性：

[cols="1h,5,1", grid=rows, frame=none]
|===
|*名称*
|*描述*
|*默认值*
|[[enabled]]enabled
|设置 `checkbox` 启用或禁用。参阅 <<enabled-state,启用状态>>。
|`true`
|[[indeterminate]]indeterminate
|设置 `checkbox` 为不确定态。参阅 <<indeterminate-state,不确定状态>>。
|`false`
|[[readOnly]]readOnly
|设置 `checkbox` 为只读模式。参阅 <<read-only-state,只读状态>>。
|`false`
|===
