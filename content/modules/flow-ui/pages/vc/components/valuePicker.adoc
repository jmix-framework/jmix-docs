== Основы

Компонент чаще всего используют для выбора сложных значений, то есть значений, выбор которых подразумевает какое-то действие.

image::visual-components/components/value-picker-basic.png[align="center",width="210"]

Пример определения `valuePicker` c `label` и двумя действиями:

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/valuepicker/value-picker-view.xml[tags=basics]
----

Чтобы создать `valuePicker`, связанный с данными, используйте атрибуты xref:vc/common-attributes.adoc#dataContainer[dataContainer] и xref:vc/common-attributes.adoc#property[property]:

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/valuepicker/value-picker-view.xml[tags=data;stepDc;layout;data-aware]
----
<1> `InstanceContainer` для сущности `Step`.
<2> Встроенный план выборки экземпляра сущности, находящегося в контейнере.
<3> Привязка компонента к контейнеру данных и свойству. Атрибут `dataContainer` содержит ссылку на контейнер данных `stepDc`, а атрибут `property` ссылается на атрибут сущности `name`.

[[actions]]
== Действия

Вы можете определить пользовательские и предопределенные действия для `valuePicker`, которые отображаются в виде кнопок справа. Это можно сделать либо в XML-дескрипторе, используя вложенный элемент `actions`, либо программно в контроллере, используя метод `addAction()`.

[TIP]
====
Чтобы добавить действие в Jmix Studio, выберите компонент в XML-дескрипторе экрана или в панели структуры *Jmix UI* и нажмите кнопку xref:studio:screen-designer.adoc#add-picker-action[Add] в панели инспектора *Jmix UI*.
====

[[custom-value-entry]]
== Обработка пользовательского ввода

`valuePicker` позволяет вам настроить его для принятия пользовательских значений.

Когда атрибут <<allowCustomValue,allowCustomValue>> установлен в `true`, пользователи могут вводить произвольные строковые значения. Это приводит к событию <<CustomValueSetEvent,CustomValueSetEvent>>.

NOTE: `valuePicker` автоматически не обрабатывает строку с пользовательским значением. Используйте событие `CustomValueSetEvent`, чтобы определить, как следует обработать введенное пользователем значение.

Пример обработки пользовательского ввода:

[source,java,indent=0]
----
include::example$/onboarding/src/main/java/com/company/onboarding/view/component/valuepicker/ValuePickerView.java[tags=notifications;vPicker;CustomValueSetEvent]
----
<1> Получите значение ввода пользователя как строку.
<2> Установите это значение в `vPicker`.

|*Описание*
|*Значение по умолчанию*
|[[allowCustomValue]]allowCustomValue
|По умолчанию пользователи не могут вручную вводить значения в поле `valuePicker`. Если вы установите атрибут `allowCustomValue` в `true`, вы включите ручной ввод. Имейте в виду, что введенное значение не будет автоматически установлено в модели данных. Для обработки пользовательского ввода используйте событие <<CustomValueSetEvent,CustomValueSetEvent>>.
