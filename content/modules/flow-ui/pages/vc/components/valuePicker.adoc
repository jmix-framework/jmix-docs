== 操作

可以给 `valuePicker` 设置自定义或预定义的操作，并作为按钮显示在右侧。
定义操作可以用 XML 中用内部的 `actions` 元素或者在控制器中用 `addAction()` 方法编程式添加。

[TIP]
====
如需在 Jmix Studio 中添加 `action`，可以在视图 XML 或者 *Jmix UI* 结构面板中选择组件，然后点击 *Jmix UI* 组件面板的 xref:studio:view-designer.adoc#add-picker-action[Add] 按钮。
====

[[custom-value-entry]]
== 自定义值

`valuePicker` 可以配置为接收自定义值。

当 <<allowCustomValue,allowCustomValue>> 属性设置为 `true` 时，用户可以手动输入自定义字符串值，并触发 <<CustomValueSetEvent,CustomValueSetEvent>>。

NOTE: `valuePicker` 不会对用户输入的自定义内容做任何处理。如果需要处理，请使用 `CustomValueSetEvent` 事件的处理方法。

示例：

[source,java,indent=0]
----
include::example$/onboarding/src/main/java/com/company/onboarding/view/component/valuepicker/ValuePickerView.java[tags=notifications;vPicker;CustomValueSetEvent]
----
<1> 以字符串类型获取用户输入。
<2> 将值设置给 `vPicker`。

== XML 属性

在 Jmix 中，所有组件都有一些作用一致的 xref:vc/common-attributes.adoc[通用属性]。
下面是 `valuePicker` 的特殊属性：

[cols="1h,5,1", grid=rows, frame=none]
|===
|*名称*
|*描述*
|*默认值*
|[[allowCustomValue]]allowCustomValue
|默认情况下，用户无法手动输入值。但如果设置 `allowCustomValue` 为 `true`，则允许用户手动输入。注意，此时手动输入的值并没有设置给数据模型，而需要通过 <<CustomValueSetEvent,CustomValueSetEvent>> 事件对输入值进行自定义处理。参阅 <<custom-value-entry, 自定义值>>。
|`false`
|===
