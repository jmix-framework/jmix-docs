= Действия компонентов списка
:page-aliases: actions/standard-actions/add-action.adoc, actions/standard-actions/create-action.adoc, actions/standard-actions/edit-action.adoc, actions/standard-actions/exclude-action.adoc, actions/standard-actions/refresh-action.adoc, actions/standard-actions/remove-action.adoc, actions/standard-actions/view-action.adoc

Действия компонентов списка работают с коллекциями сущностей, отображаемыми в компонентах xref:vc/components/dataGrid.adoc[] и xref:vc/components/treeDataGrid.adoc[]. Эти компоненты реализуют интерфейс `ListDataComponent`.

Если действие компонента списка добавлено компоненту `dataGrid` или `treeDataGrid`, оно может быть вызвано из контекстного меню компонента. Кроме того, такие действия обычно вызываются и связанными кнопками.

Далее приведены типы стандартных действий компонентов списка.

[[list_add]]
== list_add

Класс действия: `AddAction`

Добавляет существующие экземпляры сущностей в контейнер данных путем выбора их в экране списка. Может быть использовано для заполнения коллекций many-to-many.

[[list_create]]
== list_create

Класс действия: `CreateAction`

Создает новый экземпляр сущности в экране деталей сущности.

[[list_edit]]
== list_edit

Класс действия: `EditAction`

Редактирует выбранный экземпляр сущности в экране деталей сущности.

[[list_exclude]]
== list_exclude

Класс действия: `ExcludeAction`

Удаляет экземпляры сущности из контейнера данных. В отличие от действия <<list_remove,list_remove>>, `list_exclude` не удаляет выбранные экземпляры из хранилища данных. Это необходимо, например, при работе с коллекциями many-to-many.

[[list_read]]
== list_read

Класс действия: `ReadAction`

Открывает экран деталей сущности аналогично действию <<list_edit,list_edit>>, но делает все поля ввода нередактируемыми и запрещает действия сохранения. Если необходимо дать пользователям возможность переключить экран в режим редактирования, добавьте в экран действие xref:actions/view-actions.adoc#detail_enableEditing[detail_enableEditing].

[[list_refresh]]
== list_refresh

Класс действия: `RefreshAction`

Перезагружает контейнер данных, используемый компонентом списка (`dataGrid` и пр.).

[[list_remove]]
== list_remove

Класс действия: `RemoveAction`

Удаляет экземпляры сущности из контейнера данных в UI и из хранилища данных.
