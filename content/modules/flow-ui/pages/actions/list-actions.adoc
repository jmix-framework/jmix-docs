= 列表组件操作
:page-aliases: actions/standard-actions/add-action.adoc, actions/standard-actions/create-action.adoc, actions/standard-actions/edit-action.adoc, actions/standard-actions/exclude-action.adoc, actions/standard-actions/refresh-action.adoc, actions/standard-actions/remove-action.adoc, actions/standard-actions/view-action.adoc

列表组件操作支持对 xref:vc/components/dataGrid.adoc[] 或 xref:vc/components/treeDataGrid.adoc[] 中展示的实体集合进行操作。这些组件实现了 `ListDataComponent` 接口。

当 `dataGrid` 或 `treeDataGrid` 添加了列表操作后，可以从组件的右键菜单调用。通常，组件也可以通过关联的按钮进行调用。

标准列表操作有下列类型：

[[list_add]]
== list_add

操作类：`AddAction`

通过在列表视图查找实体实例的方式为数据容器添加已有的实体实例。可以用来填充多对多集合。

[[list_create]]
== list_create

操作类：`CreateAction`

在实体详情视图创建一个新的实体实例。

[[list_edit]]
== list_edit

操作类：`EditAction`

在实体详情视图编辑选择的实例。

[[list_exclude]]
== list_exclude

操作类：`ExcludeAction`

从数据容器中移除实体实例。与 <<list_remove,list_remove>> 不同，`list_exclude` 操作并不会从数据存储删除选择的实体实例。通常用在多对多集合的选择中。

[[list_read]]
== list_read

操作类：`ReadAction`

与 <<list_edit,list_edit>> 操作类似，会在详情视图打开实体，但是会将所有的字段都设置为只读状态并禁用保存操作。如果需要支持用户可以手动更换成可编辑模式，可以在视图中添加 xref:actions/view-actions.adoc#detail_enableEditing[detail_enableEditing] 操作。

[[list_refresh]]
== list_refresh

操作类：`RefreshAction`

重新加载列表组件（例如 `dataGrid`）使用的数据容器内容。

[[list_remove]]
== list_remove

操作类：`RemoveAction`

从 UI 数据容器以及数据存储中删除实体实例。
