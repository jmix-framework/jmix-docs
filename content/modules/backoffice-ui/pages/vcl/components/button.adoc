= Button

A button performs an action when a user clicks on it.

Component's XML-name: `button`

Buttons can have a caption, an icon, or both:

image::vcl/components/button.png[align="center"]

An example of defining a button with a caption and a tooltip retrieved from localized messages:

----

----

The button's caption is set using the xref:vcl/xml.adoc#caption[caption] attribute, the tooltip – using the xref:vcl/xml.adoc#description[description] attribute.

[[disableOnClick]]
If the `disableOnClick` attribute is set to `true`, the button will be automatically disabled when clicked, typically to prevent (accidental) extra clicks on a button. You can later return the button to the enabled state by invoking the `setEnabled(true)` method.

The xref:vcl/xml.adoc#icon[icon] attribute defines icon location in theme catalog or the icon name in the icon set. See xref:icons.adoc[] for more information on working with icons.

== Action

Another way of handling button clicks is associating the button with an xref:actions.adoc[Action]:

[source,xml,indent=0]
----
include::example$/ex1/src/main/resources/sample/screen/component/button/button-screen.xml[tags=action-1;action-2]
----

You can associate a button with any action defined in the window or in a table-like component (they implement `Component.ActionsHolder` interface). In case of window action, specify the action id in the `action` attribute of the button, as in the example above. In case of a component, specify the the component id and the id of the action separated by dot. In the next example a button is associated with the `create` action of the `usersTable` component:

[source, xml,indent=0]
----
include::example$/ex1/src/main/resources/sample/screen/component/button/button-screen.xml[tags=action-3]
----

Button actions can also be created programmatically in the screen controller by deriving them from xref:actions/base-action.adoc[BaseAction] class.

[NOTE]
====
If an `Action` instance is defined for a `Button`, the button will take the following properties from it: xref:vcl/xml.adoc#caption[caption], xref:vcl/xml.adoc#description[description], xref:vcl/xml.adoc#icon[icon], xref:vcl/xml.adoc#enable[enable], <<shortcut,shortcut>>, xref:vcl/xml.adoc#visible[visible]. The `caption`, `description` and `icon` properties will be imported from `Action` only if they are not set in the `Button` itself. All other listed `Action` properties have priority over the `Button` properties.

If `Action` properties are changed after the `Action` has been set for a `Button`, then `Button` properties also change accordingly, i.e. the button listens to the changes in `Action` properties. In this case, the `caption`, `description` and `icon` properties will change even if they had been initially assigned to the button itself.
====
